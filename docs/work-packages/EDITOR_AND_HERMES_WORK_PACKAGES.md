# 编辑器与 HMS/Hermes 工作包（从 MVP 到 v1）

## 主路径（产品成功条件）
- 模板约束生成（Template-constrained Generation）
  - 输出“可落地的单页 PPT HTML 模板约束规则 + schema”（支持 `.slide`、`data-index`、分页指示、键盘/触控导航、内联 CSS 与 `<script>` 约束）
- 导入（Import）
  - 支持上传/粘贴 `.html`，解析单文件 HTML，映射为可编辑的统一 slide model
- 人工精修（Manual Refinement）
  - 用户可在预览页直接微调标题、段落、卡片、列表与颜色，不破坏脚本行为
- 单页 AI 改写（Single-Slide AI Edit）
  - 用户选定某一页，提交局部改写意图，调用 HMS/Hermes 后返回 patch 并在该页内应用
- 导出（Export）
  - 导出单一 HTML（保持原单文件结构，可直接用于浏览）
- 浏览器验收（Presentation/Browser Verification）
  - 打开导出 HTML，验证 16 页切换、导航提示、键盘操作、粒子特效、布局在桌面/移动兼容

## 一致前提（全局）

## 工作包 1：MVP-0 模板约束与结构解析器（基线）
**Owner 模型建议：GPT-5.4 Mini**

**范围**
- 产出模板契约：支持 `head/style/body`、`#slides-container`、`.slide`、`data-index`、脚本分离策略。
- 实现 HTML 解析器：将输入文件解析为 `DeckModel`（幻灯片数组、元信息、样式块、脚本块、资源依赖）。
- 建立保守导入策略：导入时不破坏原 deck 的 CSS/JS；编辑器只修改已识别的 slide 片段和 `data-edit-id` 元素。安全检查以“标记风险 + 隔离预览 + 导出前验证”为主，不在导入阶段粗暴清洗可播放脚本。
- 提供“导入预检”报告（slide 数、可识别导航方法、是否包含粒子动画、是否缺失关键元素）。

**验收标准**
- 成功导入示例文件，解析出 16 页，且 `data-index` 连续且与页面顺序一致。
- 保留原样式变量与粒子画布相关 DOM/脚本行为所需 ID/Class，不丢失 `#particles`、`#page-indicator`、`#page-num` 等关键节点。
- 对无效/损坏 HTML 给出明确错误类型，不崩溃。

**风险**
- 解析器对非标准 HTML（未闭合标签、重复 ID）误判；需要做容错层。
- 样式和脚本内联过于紧耦合，粗暴清洗会导致画面失真。MVP 应优先保真，再在预览 iframe 和导出校验阶段控制风险。

## 工作包 2：MVP-1 导入器、预览器与主路径最小可用（MVP）
**Owner 模型建议：GPT-5.3 Codex Spark**

**范围**
- 搭建导入页面：上传、拖拽、粘贴导入；记录原始 HTML 与解析后模型。
- 实现演示预览层：一页一显示（或保持原始 full-screen 方案），支持键盘左右切换、点击切换、页码显示。
- 增加“恢复原样”能力（支持用户回退到导入快照）。

**验收标准**
- 可完成“导入示例文件 → 浏览所有页面 → 切换成功”的闭环。
- 16 页导入后，每页可见且切换动画可用，不会出现白屏/报错。
- 页面切换键与移动端触控行为一致，操作日志可追踪（包含导入时间戳和当前页）。

**风险**
- 现有 JS 与自实现 UI 的事件冲突（全局 keydown 被重复监听）。
- 多终端字体与高分辨率下布局溢出，影响可编辑区域交互。

## 工作包 3：MVP-2 手工精修工具链（Manual Refinement）
**Owner 模型建议：GPT-5.4 Mini + GPT-5.3 Codex Spark**

**范围**
- 在单页中提供局部编辑模式（文本块、列表块、卡片块、颜色/字号微调）并保留渲染脚本完整性。
- 变更保存为可回放的 `operation log`（`slideId + patch + timestamp`）。
- 提供预览差异：当前页即时对比“原始 vs 编辑后”。
- 支持草稿自动保存与恢复，防止误触后丢失修改。

**验收标准**
- 用户可在不进入代码视图的情况下完成一页至少 5 处不同类型内容修改。
- 页面切换前后不丢失已编辑内容，编辑后的 DOM 可再次导出并在浏览器正常显示。
- 所有编辑动作可导出“可追踪日志”（用于回滚/对账）。

**风险**
- 过度自动清洗导致用户手动样式丢失；需把“仅改文本/结构”与“改脚本/样式”隔离。
- 复制粘贴富文本带入外链、脚本、不可见控制字符，需要过滤。

## 工作包 4：MVP-3 单页 AI 改写接入 HMS/Hermes
**Owner 模型建议：更强模型（Hermes 默认路由模型，必要时用更强模型降噪）**

**范围**
- 统一设计 HMS/Hermes 调用边界（只对当前选中页）：
  - `POST /api/hms/slide-edit`
  - 请求：`slideIndex`、`selectedHtml`、`editIntent`、`constraints`（保留粒子、导航、脚本行为）、`templateId`
  - 响应：`updatedSlideHtml`、`patch`、`warnings`、`modelUsed`、`costTokens`
- 在“单页”粒度生成改写草案并预览差异，确认后提交。
- 仅允许修改当前页 DOM 片段，不允许跨页重排与全局资源注入。

**验收标准**
- 对某一页发起 3 种不同意图（文案重写、结构微调、风格微调）均成功返回结果且可一键应用。
- 成功率 >= 90%（在样本 3 次内手工测试不低于 90% 生效）。
- 改写后不影响该页的导航、粒子、键盘事件和页面索引。

**风险**
- 提示词未严格约束单页上下文可能触发跨页污染。
- Hermes 响应延迟导致交互卡顿，需超时与 loading 状态处理。

## 工作包 5：v0.8 离线降级与稳定性（HMS 不可用时）
**Owner 模型建议：GPT-5.4 Mini**

**范围**
- 定义最小本地回退：若 Hermes/HMS 不可用，自动退化为“本地规则改写”：
  - 文案本地优化（同义词替换、长度压缩、术语统一）
  - 常用排版调整（字体大小/间距、对齐）
  - 仅允许纯文本与纯样式级修改，不调用远端模型
- UI 明确提示“离线模式”；允许用户先缓存改写意图，恢复后再提交。
- 失败任务落库（重试队列）和可见状态。

**验收标准**
- Hermes 不可达时，用户仍可完成手工编辑与导出流程。
- 本地回退不会改写脚本块和粒子系统；导入后的页面稳定可播。
- 任务重试成功率在网络恢复后可回放，且未丢失用户编辑文本。

**风险**
- 本地回退能力有限，复杂设计意图无法达成；需清楚提示用户预期。
- 缓存重试与幂等性处理错误可能导致重复应用同一改写。

## 工作包 6：v1 发布与验收（Export + Browser Verification）
**Owner 模型建议：GPT-5.3 Codex Spark（打包） + 更强模型（验收抽检用 prompt）**

**范围**
- 实现导出器：输出标准单文件 HTML（保持内联样式和脚本，文件名含版本号+时间戳），并支持“保持脚本完整导出”和“安全脚本导出”两档。
- 增加 `playwright`/脚本级验收：打开导出文件，验证 16 页、首尾页切换、导航提示、粒子 canvas 渲染、缩放窗口自适应。
- 输出验收报告：通过项/失败项、耗时、失败截图、关键日志。

**验收标准**
- 端到端闭环可重放：模板生成 → 导入 → 手工精修 → 单页AI改写 → 导出 → 浏览器打开验证，全链路通过率 >= 95%（连续 3 次手测）。
- 导出文件可本地双击打开/浏览器打开，具备完整演示可播放体验。
- 生成的验收报告可追溯到每个工作包与模型调用记录。

**风险**
- 导出文件体积膨胀影响加载速度。
- 不同浏览器对 `canvas`、`requestAnimationFrame` 的差异导致特效表现波动。

## Hermes/HMS/API 边界与模型路由建议
- 建议路由：
  - 模板提取、结构化解析、规则校验：**GPT-5.4 Mini**
  - 页面交互实现、状态管理、导出验收脚本：**GPT-5.3 Codex Spark**
  - 复杂改写语义、重写策略、降噪提示词：**更强模型（Fallback 到 GPT-5.4 Mini）**
- 约束要求（必须）：
  - 单页改写仅接受当前页 HTML 片段。
  - 禁止修改 `script`、`on*`、`<iframe>` 等危险入口。
  - 返回结果必须带 `patch` + 变更摘要 + 风险标签（样式重排/文本长度变化）。
- 降级策略：
  - 5 秒超时或 Hermes 401/429/5xx 时，自动切本地回退并标记任务为“待云端重试”。
  - 恢复网络后只重放未完成队列，不自动改写历史页。
